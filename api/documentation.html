Introduction

The Company Data API provides programmatic access to company recruitment information. This API allows you to retrieve company details, search for companies based on various criteria, and access industry and company size information.

Base URL

All API requests should be made to:

https://yourusername.github.io/company-data-api/api/

Getting an API Key

To use the Company Data API, you need an API key. For demonstration purposes, you can use the following demo key:

demo-key-123456

For production use, please contact us to obtain a dedicated API key with higher rate limits.

Authentication

All API requests require authentication using an API key. You should include your API key in each request as a query parameter:

?api_key=YOUR_API_KEY

Example

https://yourusername.github.io/company-data-api/api/?endpoint=companies&api_key=demo-key-123456

If you don't provide a valid API key, you'll receive a 401 Unauthorized response.

Code Examples

JavaScript

// Get all companies
async function getCompanies() {
  const apiKey = 'demo-key-123456';
  const response = await fetch(`https://yourusername.github.io/company-data-api/api/?endpoint=companies&api_key=${apiKey}`);
  const data = await response.json();
  
  if (data.status === 200) {
    console.log('Companies:', data.data);
    return data.data;
  } else {
    console.error('Error:', data.error);
    return null;
  }
}

Python

import requests

def get_companies():
    api_key = 'demo-key-123456'
    url = f'https://yourusername.github.io/company-data-api/api/?endpoint=companies&api_key={api_key}'
    
    response = requests.get(url)
    data = response.json()
    
    if data.get('status') == 200:
        print(f"Companies: {len(data['data'])} found")
        return data['data']
    else:
        print(f"Error: {data.get('error')}")
        return None

Java

import java.net.HttpURLConnection;
import java.net.URL;
import java.util.Scanner;
import org.json.JSONObject;

public class CompanyApiClient {
    public static JSONObject getCompanies() {
        try {
            String apiKey = "demo-key-123456";
            URL url = new URL("https://yourusername.github.io/company-data-api/api/?endpoint=companies&api_key=" + apiKey);
            
            HttpURLConnection conn = (HttpURLConnection) url.openConnection();
            conn.setRequestMethod("GET");
            conn.connect();
            
            int responseCode = conn.getResponseCode();
            
            if (responseCode == 200) {
                StringBuilder response = new StringBuilder();
                Scanner scanner = new Scanner(url.openStream());
                
                while (scanner.hasNext()) {
                    response.append(scanner.nextLine());
                }
                scanner.close();
                
                JSONObject jsonResponse = new JSONObject(response.toString());
                return jsonResponse;
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
        
        return null;
    }
}

Kotlin

import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext
import org.json.JSONObject
import java.net.URL

suspend fun getCompanies(): JSONObject? {
    return withContext(Dispatchers.IO) {
        try {
            val apiKey = "demo-key-123456"
            val url = URL("https://yourusername.github.io/company-data-api/api/?endpoint=companies&api_key=$apiKey")
            
            val connection = url.openConnection()
            connection.connectTimeout = 5000
            connection.readTimeout = 5000
            
            val response = connection.getInputStream().bufferedReader().use { it.readText() }
            JSONObject(response)
        } catch (e: Exception) {
            e.printStackTrace()
            null
        }
    }
}

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of requests. Error responses include an error message to help you understand what went wrong.

Common Error Codes

Status Code Description
400 Bad Request - The request was invalid or missing required parameters
401 Unauthorized - API key is missing or invalid
404 Not Found - The requested resource was not found
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Something went wrong on the server

Error Response Format

{
  "error": "Error message describing what went wrong",
  "status": 401
}

Rate Limits

To ensure the stability of the API, rate limits are enforced on all API keys. The demo key has a limit of 100 requests per hour.

If you exceed your rate limit, you'll receive a 429 Too Many Requests response. The rate limit resets every hour.

Rate Limit Response

{
  "error": "Rate limit exceeded",
  "status": 429
}

For higher rate limits, please contact us to obtain a dedicated API key for your application.
